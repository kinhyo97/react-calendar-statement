Calendar API Spec

Base URL: http://localhost:4000

Auth: Bearer token. Login response token is stored by the frontend in localStorage.

Database: SQLite file at api-server/data/calendar.db.

Data Models

Model Shape
AuthUser { id: string, email: string, name: string }
AuthSession { token: string, user: AuthUser }
CalendarEvent { id: string, userId: string, title: string, startDate: string, endDate: string, categoryId: string, memo?: string }
EventInput { title: string, startDate: "YYYY-MM-DD", endDate: "YYYY-MM-DD", categoryId: string, memo?: string }
CalendarCategory { id: string, userId: string, name: string, color: "#RRGGBB" }
CategoryInput { name: string, color: "#RRGGBB" }

Authentication

POST/auth/login

Logs in an existing user. If the email does not exist, a local user is created.

Request

{
  "email": "demo@example.com",
  "password": "password"
}

Response 200

{
  "token": "uuid-token",
  "user": {
    "id": "uuid-user",
    "email": "demo@example.com",
    "name": "demo"
  }
}
POST/auth/logout

Deletes the current session token.

Headers

Authorization: Bearer <token>

Response 200

{ "ok": true }
GET/me

Returns the authenticated user for the bearer token.

Headers

Authorization: Bearer <token>

Response 200

{
  "id": "uuid-user",
  "email": "demo@example.com",
  "name": "demo"
}

Categories

GET/categories

Returns editable categories owned by the authenticated user. Default categories are created automatically.

Headers

Authorization: Bearer <token>

Response 200

[
  {
    "id": "work",
    "userId": "uuid-user",
    "name": "업무",
    "color": "#2f8fe9"
  }
]
PUT/categories/:id

Updates a category name or color for the authenticated user.

Headers

Authorization: Bearer <token>

Request

{
  "name": "회사",
  "color": "#2563eb"
}

Events

GET/events

Returns events owned by the authenticated user, sorted by date.

Headers

Authorization: Bearer <token>

Response 200

[
  {
    "id": "uuid-event",
    "userId": "uuid-user",
    "title": "프로젝트 회의",
    "startDate": "2026-05-16",
    "endDate": "2026-05-18",
    "categoryId": "work",
    "memo": "상태관리 비교"
  }
]
POST/events

Creates an event for the authenticated user.

Headers

Authorization: Bearer <token>

Request

{
  "title": "프로젝트 회의",
  "startDate": "2026-05-16",
  "endDate": "2026-05-18",
  "categoryId": "work",
  "memo": "상태관리 비교"
}

Response 201

{
  "id": "uuid-event",
  "userId": "uuid-user",
  "title": "프로젝트 회의",
  "startDate": "2026-05-16",
  "endDate": "2026-05-18",
  "categoryId": "work",
  "memo": "상태관리 비교"
}
PUT/events/:id

Updates an event owned by the authenticated user.

Headers

Authorization: Bearer <token>

Request

{
  "title": "수정된 일정",
  "startDate": "2026-05-17",
  "endDate": "2026-05-20",
  "categoryId": "study",
  "memo": "TanStack Query mutation"
}
DELETE/events/:id

Deletes an event owned by the authenticated user.

Headers

Authorization: Bearer <token>

Response 200

{ "id": "uuid-event" }

Error Responses

Status Meaning
400 Invalid request body.
401 Missing token, invalid token, or wrong password.
404 Route or event not found.
500 Unexpected server error.
